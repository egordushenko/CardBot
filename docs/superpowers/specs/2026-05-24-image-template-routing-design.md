# Шаблоны генерации изображений GPT Image 2

## Цель

Заменить одинаковый короткий промпт для всех изображений на последовательность из семи коммерческих ролей слайдов. Для одежды используется специализированный набор из `docs/Image_Promts.md`, для остальных товаров - универсальный набор из `docs/universal_marketplace_templates_gpt_image_2.txt`.

## Текущий контур

- `llm.generate_image_prompts()` сейчас создает `N` одинаковых `ImageConcept`.
- `image_generator._build_single_image_prompt()` добавляет общий текст о референсном коллаже и одном результате 3:4.
- Генерация текста карточки и план изображений в режиме `text_and_images` запускаются параллельно.
- `category_profile` уже определяется до запуска обоих задач, но сейчас не передается в генерацию image-plan.

## Решение

### Выбор набора

`generate_image_prompts()` получает `category_profile` и выбирает набор ролей по категории:

- одежда: маркеры категории одежды или специализированной одежды (`одежда`, `рашгард`, `футболка`, `лонгслив`, `худи`, `платье` и аналогичные);
- прочие товары: универсальный набор.

Определение опирается только на `category_profile`. Если профиль отсутствует или не относится к одежде, применяется универсальный набор.

### Последовательности ролей

Для одежды набор зависит от выбранного количества изображений:

- `1`: инфографика преимуществ.
- `3`: инфографика преимуществ; крупный план деталей; lifestyle на модели без фиксированного ракурса.
- `5`: инфографика преимуществ; крупный план деталей; lifestyle на модели спереди; lifestyle на модели сзади; главная карточка.
- `7`: полный набор: главная карточка; инфографика преимуществ; lifestyle на модели спереди; lifestyle на модели сзади; крупный план деталей; размеры и посадка; сценарий использования.

Слайд крупного плана для одежды мягко рекомендует один небольшой блок инфографики, поясняющий показанную деталь или усиливающий ее подачу.

Для прочих товаров:

1. Главная карточка.
2. Инфографика преимуществ.
3. Крупный план и детали товара.
4. Сценарий использования.
5. Характеристики, размеры и параметры.
6. Комплектация.
7. Ключевая функция, технология или особенность.

Для универсальных товаров при запросе `1`, `3`, `5` или `7` изображений используются первые `N` ролей набора. Для одежды используются определенные выше наборы. Каждый concept остается отдельным запросом `1 request = 1 image`.

### Содержание промптов

Шаблоны будут встроены в код как стабильные константы и сокращены только там, где это требуется существующим контрактом.

Каждый итоговый промпт получает:

- роль конкретного слайда;
- описание товара пользователя;
- определенную категорию, если она есть;
- пожелания пользователя и выбранный style preset;
- правила сохранения идентичности товара по входному коллажу;
- требование создать одно изображение 3:4, а не коллаж;
- запрет придумывать неподтвержденные свойства, комплектность, размеры, технологии и надписи.

Поля шаблонов вида преимуществ, характеристик и комплектности не заполняются данными из сгенерированной текстовой карточки. Вместо этого модель получает инструкцию выводить только явно указанные пользователем или визуально подтверждаемые факты.

### Поток данных

1. Бот определяет `category_profile` существующим механизмом.
2. Тот же профиль передается в `_generate_image_prompts_for_batch()` и далее в `generate_image_prompts()`.
3. `generate_image_prompts()` выбирает clothing/universal-набор и формирует разные `ImageConcept`.
4. `image_generator` добавляет единый технический контракт коллажа, формата и single-image output.
5. Все concept-запросы продолжают выполняться параллельно с текущим лимитом.

## Границы

- Не добавляется еще один LLM-вызов для подготовки промптов.
- Не требуется ждать завершения генерации текстовой карточки.
- Не меняется сборка коллажа, стоимость одного изображения или параллельность.
- Не меняется пользовательский flow выбора количества изображений, стиля и пожеланий.

## Проверка

- Тест: одежда получает согласованный набор ролей для `1`, `3`, `5` и `7` изображений.
- Тест: слайд деталей одежды содержит мягкую рекомендацию блока инфографики.
- Тест: обычный товар получает универсальные роли.
- Тест: пожелания/style preset включаются в каждый concept.
- Тест: `category_profile` передается из обоих image flows.
- Регрессия: итоговый запрос к GPT Image 2 остается single-image 3:4 с одним входным коллажем.
- Полный запуск `pytest`.
