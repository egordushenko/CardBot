# Marketplace rules for CardBot

Дата проверки: 2026-05-21.

Документ фиксирует правила, которые CardBot использует в промптах и постобработке. Это не замена официальной документации маркетплейсов: перед крупными изменениями правила нужно перепроверять.

## Ozon

Источник: https://docs.ozon.com/global/products/requirements/product-info/naming-requirements/

### Название

- Максимальная длина названия: 200 символов.
- Максимальная длина одного слова: 27 символов.
- Рекомендуемая схема: бренд + тип товара + модель + важные характеристики.
- Тип товара в названии должен соответствовать выбранной категории.
- Важные характеристики лучше указывать ближе к началу.
- Нельзя перегружать название характеристиками и синонимами.
- Запрещены символы: `™`, `©`, `®`, `[`, `]`, `=`, `\`, `«`, `»`.
- Запрещены рекламные и рискованные формулировки: `акция`, `распродажа`, `скидка`, `только сегодня`, `лучший товар`, `уникальный товар`, `аналог`, `подделка`, `копия`, `copy`, `реплика`, `1:1`, `по мотивам`, `оригинал`, `original`, `подлинный`.
- Нельзя писать обычные слова транслитом или капсом. Исключение: аббревиатуры и официальные названия производителя.

### Описание

Источник: https://docs.ozon.com/global/products/requirements/product-info/product-description/

- Поле называется `Аннотация`.
- Описание должно давать покупателю понятное представление о товаре, преимуществах и вариантах использования.
- Нельзя добавлять контакты, ссылки, соцсети, телефоны, отсылки к другим источникам.
- Нельзя добавлять большое количество нерелевантных ключей и перечислений.
- Нельзя использовать рекламные акции, цены, обещания вроде `лучший товар`, а также слова про копии, реплики и `original`.
- Для CardBot целевой размер описания Ozon пока оставляем 600-900 символов, потому что это продуктовый UX-лимит: текст должен быть короче WB и удобен для копирования.

### Хештеги

Источник: https://docs.ozon.com/global/products/requirements/product-info/hashtags/

- Максимум 30 хештегов на товар.
- Каждый хештег начинается с `#`.
- Разрешены буквы, цифры и нижнее подчёркивание.
- Максимальная длина одного хештега: 30 символов.
- Хештеги разделяются пробелом.
- Нельзя добавлять одинаковые по смыслу хештеги.
- Нельзя дублировать характеристики, которые уже есть в карточке, например бренд или тип товара.

### Характеристики

Источник: https://docs.ozon.com/global/products/requirements/product-info/product-characteristics/

- Набор характеристик зависит от категории товара.
- Для CardBot результат остаётся в формате `Ключ: Значение`, одна характеристика на строку.
- Значения нужно брать из описания пользователя. Неизвестные параметры нельзя выдумывать.

## Wildberries

Источник: https://seller.wildberries.ru/instructions/ru/by/material/how-to-create-a-product-profile-rb

### Название

- Максимальная длина названия: 60 символов.
- Для части категорий название формируется автоматически из типа товара и характеристик.
- Для CardBot целевая структура: основной запрос + ключевая характеристика + преимущество.
- Для WB не указываем пол и цвет в названии, если их можно вынести в характеристики.

### Описание

- Для CardBot целевой размер: 800-1200 символов.
- Структура: короткий смысловой блок, преимущества, сценарии применения.
- Ключевые слова встраиваются естественно, без переспама.

### Ключевые слова

- Для CardBot поле остаётся отдельным SEO-блоком: 12-15 запросов через запятую.
- Без дублей, мусора, стоп-слов и искусственных повторов.

### Характеристики

- Формат CardBot: `Ключ: Значение`, одна характеристика на строку.
- Материал, цвет, размер, пол, совместимость, тип и другие параметры берутся из описания пользователя.
- Неизвестные параметры нельзя выдумывать.

## Runtime post-processing

CardBot дополнительно нормализует результат после ответа модели:

- обрезает название WB до 60 символов;
- обрезает название Ozon до 200 символов и слова в нём до 27 символов;
- удаляет из Ozon title запрещённые символы и рекламные/рисковые фразы;
- удаляет из Ozon description ссылки, контакты, телефоны, рекламные/рисковые фразы;
- нормализует Ozon hashtags: добавляет `#`, убирает дубли, пропускает теги длиннее 30 символов;
- очищает характеристики от служебных хвостов модели и строк без формата `Ключ: Значение`.
