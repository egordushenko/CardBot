# Ozon dataset collection plan

Цель: довести `data/ozon_cards_dataset.json` до 200 референсных карточек Ozon с нормальным покрытием категорий.

Текущий статус на момент планирования:

- собрано: 50 карточек;
- нужно добрать: 150 карточек;
- минимальный рейтинг: 4.7;
- минимум отзывов: 1000;
- источник категорий: `data/ozon_categories.json`.

## Критерии карточки

Брать карточку, если:

- `rating >= 4.7`;
- `review_count >= 1000`;
- есть название;
- есть описание;
- есть характеристики;
- есть хештеги;
- карточка не дублирует уже собранный `product_id` или `source_url`.

Не брать:

- 10 однотипных товаров одного бренда;
- карточки без характеристик;
- карточки без описания;
- карточки, где текст явно слабый, мусорный или состоит из одних ключей.

## Как проверять прогресс

После каждой партии запускаем:

```powershell
py -3.11 tools\ozon_dataset_audit.py --dataset data\ozon_cards_dataset.json --output-md data\ozon_dataset_report.md
```

Смотрим:

- `Invalid cards` должен быть 0;
- дубли `product_id` и `source_url` должны быть 0;
- категории с самым большим `Missing` идут в следующий сбор;
- среднее количество характеристик должно расти или хотя бы не падать;
- среднее количество хештегов должно быть в реалистичном диапазоне, не 0-1.
