# WB generation patterns from reference dataset

Основа анализа:
- `data/wb_cards_dataset.json`: 772 валидные карточки.
- `data/wb_dataset_analysis.json`: агрегаты по 13 верхним категориям.
- `docs/wb_rules.md`: официальные правила WB, зафиксированные отдельно.

## Главный вывод

Текущий WB-промпт в `prompts.py` стоит усилить не общими SEO-советами, а категорийными правилами. Популярные карточки WB дают три сильных сигнала:

- Название короткое: среднее 40.5 символа, p90 56, жесткий потолок 60.
- Описание длиннее текущего нижнего ожидания: среднее 1282 символа, p50 1224.
- Характеристики важнее, чем сейчас заложено в промпте: среднее 14.6 поля, p75 17, p90 24.

## Название

### Что изменить в промпте

Текущая формула "основной запрос + характеристика + преимущество" слишком рекламная. Для WB лучше:

`тип товара + уточнение категории + ключевой параметр выбора + модель/размер/количество при наличии`

Не писать в названии абстрактные преимущества: `удобный`, `качественный`, `лучший`, `идеальный`, `премиум`, `топ`, `хит`. В референсах название чаще похоже на сухую поисковую строку.

### Пол и цвет

Текущий запрет "не указывай пол и цвет" нужно ослабить.

Правило:
- Пол указывать, если это часть покупательского запроса: `женские кроссовки`, `мужская футболка`, `детский костюм`.
- Цвет указывать, если он реально влияет на выбор и дан пользователем: одежда, обувь, смартфоны, техника, декор.
- Не дублировать цвет и пол насильно, если они не нужны или уже очевидны из категории.

### Целевые длины

- Общий таргет: 35-55 символов.
- Одежда/обувь/аксессуары: 25-50 символов.
- Электроника/техника/стройка: 40-60 символов.
- Продукты/здоровье/красота: 35-60 символов.

### Категорийные схемы названий

Женщинам:
- `тип + крой/посадка + материал/сезон + назначение`
- Пример схемы: `Футболка оверсайз хлопковая летняя`
- Не раздувать: бренд и цвет только если они заданы или критичны.

Мужчинам:
- `тип + базовость/крой + материал/рукав/сезон`
- Пример схемы: `Футболка однотонная базовая хлопок`

Обувь:
- `тип обуви + пол + сезон/материал + подошва/застежка`
- Пример схемы: `Кроссовки женские на высокой подошве`

Детям:
- `тип + возраст/назначение + комплектность`
- Пример схемы: `Боди для новорожденных 3 шт`

Дом:
- `тип + зона применения + материал/свойство + размер/количество`
- Пример схемы: `Коврик для ванной противоскользящий 50х80`

Электроника:
- `тип + модель + ключевая спецификация + цвет`
- Пример схемы: `Смартфон Z10 Lite 8+256GB 6000 мАч`

Бытовая техника:
- `тип + назначение + мощность/объем/программа + модель`
- Пример схемы: `Мультиварка 5 л с 20 программами`

Красота:
- `тип средства + действие/зона + объем/количество`
- Пример схемы: `Спрей для роста волос 100 мл`

Здоровье:
- `форма + назначение + активный компонент + объем/количество`
- Пример схемы: `Спрей для горла с прополисом 30 мл`
- Не писать лечебные обещания, если пользователь их явно не дал.

Продукты питания:
- `тип + вкус/форма + количество/вес`
- Пример схемы: `Чай черный в пирамидках 20 шт`

Строительство и ремонт:
- `тип + материал + размер + комплектация`
- Пример схемы: `Ванна акриловая 170х70 с каркасом`

Игрушки:
- `тип + механика/назначение + персонаж/размер`
- Пример схемы: `Игрушка антистресс мягкая 18 см`

## Описание

### Что изменить в промпте

Текущий диапазон 800-1200 символов нормальный как базовый, но его лучше сделать категорийным.

Рекомендуемые диапазоны:
- Одежда, обувь, детские, аксессуары: 700-1200.
- Дом, игрушки, техника, электроника, стройка: 900-1400.
- Красота и здоровье: 1000-1800, но без медицинских обещаний.
- Продукты питания: 600-1000.

### Структура описания

Лучший базовый шаблон:

1. Первый абзац: что это за товар и главная польза в 1-2 предложениях.
2. Второй абзац: конкретные свойства из входа пользователя: материал, состав, размер, объем, комплект, совместимость, сезон, уход.
3. Третий абзац: сценарии применения и кому подойдет.
4. Финальная фраза: аккуратное резюме без призыва "купите сейчас".

Не заставлять модель писать маркированный список внутри описания. Для WB описание лучше выглядит как связный текст с короткими абзацами. Списки можно использовать только если пользователь просит "коротко" или товар технический.

### Запрещенные обороты

Убрать из генерации:
- `Представляем вашему вниманию`
- `Идеальный выбор для каждого`
- `Незаменимый помощник`
- `Высокое качество`
- `Премиальное качество`
- `Лучший подарок`
- `Хит продаж`
- `Успейте купить`
- `Скидка`, `акция`, `распродажа`

### Категорийные акценты описания

Одежда:
- посадка, ткань, сезон, сочетания, уход;
- не обещать универсальный размер, если размера нет.

Обувь:
- сезон, подошва, фиксация, материал верха/стельки/подкладки, повседневные сценарии.

Дом:
- зона применения, материал, размер, уход, устойчивость/влагостойкость/хранение.

Электроника:
- модель, память, экран, батарея, гарантия, комплектация, совместимость.

Бытовая техника:
- объем, мощность, программы, сценарии кухни/дома, комплектация, уход.

Красота:
- зона применения, действие, тип кожи/волос, активы, объем, способ применения;
- не писать лечебные обещания.

Здоровье:
- форма выпуска, состав, назначение, ограничения, условия хранения;
- избегать обещаний лечения и диагнозов.

Продукты:
- вкус, формат, состав, вес, количество, способ приготовления, хранение.

Стройка:
- материал, размер, монтаж, комплектация, совместимость, гарантия.

Игрушки:
- возраст/сценарий, материал, размер, развитие/антистресс, безопасность без неподтвержденных сертификатов.

## Ключевые слова

WB не использует хэштеги в карточках так, как Ozon. В датасете поле `hashtags` везде пустое.

Для бота:
- `keywords` оставить как SEO-фразы через запятую.
- Не генерировать `hashtags` для WB.
- 10-15 фраз достаточно.
- Формула: 4-5 широких запросов + 5-7 средних + 2-3 длинных уточняющих.

Пример для коврика:
`коврик для ванной, коврик в ванную, противоскользящий коврик, коврик для туалета, коврик 50х80, мягкий коврик для ванной, коврик для ванной комнаты`

## Конкретные изменения для бота

P0 - переписать WB_SYSTEM_PROMPT:
- убрать абсолютный запрет на пол и цвет в названии;
- заменить рекламную формулу названия на сухую WB-формулу;
- добавить категорийные схемы названий;
- увеличить таргет характеристик до 12-18;
- для электроники/стройки/техники разрешить 18-28 характеристик;
- явно запретить хэштеги для WB.

P1 - добавить отдельные WB category profiles:
- не смешивать Ozon и WB в `data/category_profiles.json`;
- создать `data/wb_category_profiles.json` из `data/wb_dataset_analysis.json`;
- в `llm.py` подставлять профиль по marketplace.

P1 - добавить пост-валидацию WB:
- `title <= 60`;
- `hashtags` отсутствуют или пустые;
- `description` без рекламных слов;
- `characteristics` минимум 8 строк;
- для одежды проверять `Состав`, `Цвет`, `Пол`;
- для электроники проверять `Модель`, `Гарантийный срок`, `Комплектация`.

P2 - улучшить category detector:
- текущие WB-профили работают по верхним категориям, но лучше учитывать подкатегорию: `Платья`, `Коврики`, `Видеокарты`, `Витамин D`, `Ванны`.
- подкатегория должна менять поля характеристик и схему названия.
